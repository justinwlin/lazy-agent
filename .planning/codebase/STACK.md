# Technology Stack

**Analysis Date:** 2026-01-15

## Languages

**Primary:**
- Bash - All automation scripts (`setup.sh`, `doctor.sh`, `update.sh`, `dotfiles/install.sh`)
- Markdown - Documentation and AI guidance (`steps/*.md`, `AGENTS.md`, `.claude/skills/*/SKILL.md`)

**Secondary:**
- JSON - Configuration (`config.json`, `.devcontainer/devcontainer.json`, `.claude/settings.local.json`)

## Runtime

**Environment:**
- macOS - Primary target platform (Homebrew, Ghostty, Karabiner)
- Node.js 20 - Specified in `.devcontainer/devcontainer.json`
- zsh - Shell runtime (configured via `dotfiles/.config/ghostty/config`)
- Docker - Dev container support via `.devcontainer/`

**Package Manager:**
- Homebrew - Primary macOS package manager (`steps/01-prerequisites.md`)
- npm - Node.js packages for Claude Code (`@anthropic-ai/claude-code`)
- Go - For multi-agent tools like Gastown and Beads

## Frameworks

**Core:**
- None (shell-based CLI orchestration, no application framework)

**Testing:**
- None (uses manual verification via `doctor.sh` health check script)

**Build/Dev:**
- Dev containers - VS Code devcontainer support (`.devcontainer/devcontainer.json`)
- tmux - Terminal multiplexer with custom configuration (`dotfiles/.tmux.conf`)

## Key Dependencies

**Critical:**
- Claude Code (`@anthropic-ai/claude-code`) - AI coding assistant, core to onboarding
- Homebrew - Package manager for all tool installations
- tmux 3.0+ - Terminal multiplexer with vim-style bindings, FZF integration, LazyGit

**Infrastructure:**
- Ghostty - GPU-accelerated terminal emulator (`dotfiles/.config/ghostty/config`)
- Oh My Zsh - Zsh configuration framework
- Powerlevel10k - Advanced zsh prompt theme

**Multi-Agent (Optional):**
- Gastown - Multi-agent workspace coordination (`steps/07-gastown.md`)
- Beads - Git-backed issue tracking with Linear sync (`steps/08-linear-and-mcp.md`)

## Configuration

**Environment:**
- `config.json` - User preferences (generated by `setup.sh`)
- `~/.zshrc.local` - Environment variables (LINEAR_API_KEY, ANTHROPIC_API_KEY)
- No `.env` files used - configuration is shell-based

**Build:**
- No build configuration - shell scripts execute directly

## Platform Requirements

**Development:**
- macOS (primary), Linux (partial support via devcontainer)
- Homebrew required for package installation
- No external services required for basic setup

**Production:**
- Not applicable - this is a development environment setup tool
- Distributed as git repository, cloned locally
